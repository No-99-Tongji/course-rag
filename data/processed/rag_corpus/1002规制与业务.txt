一、第十章第二节：规制
规制是通过规则或限制来控制行为或控制过程。
《牛津英文词典》将规制定义为“规制的行为或事实”，将 to regulate 界定为“控制、支配或管理”。
规制以解决市场失灵、维持市场经济秩序为目的，基于规则对市场主体的经济活动，以及伴随其经济活动产生的社会问题进行干预和控制。

二、软件项目与规制的关系
每个软件项目或者每项软件业务都需要遵守或受制于一定的规则。
在软件项目或软件业务的经济分析中，需要正确分析和评价软件与环境、社会与治理之间的关系。
环境、社会与治理简称 ESG，ESG 来源于 Environmental、Social、Governance 这三个英文单词的首字母。
常见的软件知识产权、软件工程伦理、数字经济与监管等软件活动都属于规制的范畴。

三、软件业务规制的三个领域
软件业务规制从其研究对象看，涵盖三个领域：
经济性规制
社会性规制
竞争

四、经济性规制
经济性规制是指对于软件组织或软件项目绩效、价格、投资及产出等经济问题的规则、标准和执行程序。
这是软件工程经济学最常用的领域，也是本课程讲解的主要内容。

五、社会性规制
社会性规制包括教育、健康、安全、环境、反歧视等领域的规则。
这类规则没有明显的经济目标，但确实会产生经济效果、成本和收益。
分析和评价特定社会性规则对于软件工程活动或软件项目会产生怎样的经济及社会影响，是软件工程师必须知晓的基础知识之一。

六、竞争
竞争是指软件行业的垄断、滥用垄断权力、合资、兼并等可能赋予软件组织以过度的市场权力。
竞争规则或法律支持市场的决定性作用，并对阻碍竞争和滥用权力的行为给予惩戒。

七、案例一：大众点评诉百度侵权案
网络的极强交互性使得用户在浏览网上触手可及的信息的同时，用户的信息也正在被别人收集、分享甚至使用。
该案件概要是：百度大量抓取大众点评网用户的公开点评信息，并在百度产品中直接呈现。
大众点评网遂将百度诉诸法院。

八、大众点评诉百度案的数据采集方式与核心问题
数据的采集方式：百度基于网络公开的用户数据，采用爬虫技术抓取相应的数据。
该案的核心问题是：针对网站上用户公开信息的收集和使用规则应该如何界定。

九、大众点评诉百度案的分析角度
分析的角度：认定原告和被告为存在竞争关系的竞争者，基于竞争法展开分析。
具体行为是：百度通过网络爬虫抓取数据后，将他方网站公开的用户点评信息直接呈现在百度产品搜索结果之中。
相关市场：大众点评网与百度在为用户提供商户信息和点评信息方面的服务模式相似，它们争夺同样的网络用户群体，因此具有竞争关系。

十、大众点评诉百度案的落脚点与判决结果
该案的落脚点是：百度大量全文展示来自大众点评网的信息，使用方式超过必要限度，未遵循最少必要原则，因而不具有正当性。
判决结果：法院终审判决被抓取方获胜，爬虫方落败。

十一、案例二：谷歌与甲骨文侵权案
谷歌公司与甲骨文公司之间围绕上万行代码展开的诉讼历时十年，于 2021 年 4 月 5 日结束。
当天美国最高法院判定，谷歌对计算机编程语言 Java 部分代码的使用符合合理使用原则，无需为此向甲骨文支付费用。

十二、API 的含义及案件影响
API，即应用程序编程接口，是一批预先写好的代码。
通过 API，程序员可以使用大量预先编写好的代码库来执行复杂的任务，而无需从头开始去写代码。
由于 API 被广泛应用于全球软件行业之中，此案因此受到整个信息行业的关注。
微软、IBM 等公司均向美国最高法院提交了他们对本案的法律意见。
这些科技公司在其提交给法院的文书中表示，之前下级法院的裁决会产生灾难性后果，会消灭打造创新生态所必要的喘息空间。

十三、谷歌与甲骨文案的判决意见
美国最高法院大法官 Stephen Breyer 撰写了法庭多数意见。
判决认可了下级法院关于合理使用审查的意见，认为合理使用兼具事实问题和法律问题两面性。
陪审团仅认定事实问题，但能否根据事实得出合理使用的结论，仍然需要由法官决定。
因此，下级法院推翻陪审团裁决的行为，与宪法第七修正案“由陪审团裁决的事实不得重新审查”的规定并不冲突。
谷歌要求由陪审团对合理使用做最终决定的主张没有法律依据。
Breyer 假定 API 受到版权保护，进而直接对衡量合理使用的四个因素进行审查。

十四、合理使用因素一：受版权保护作品的性质
Breyer 认为 API 仅作为声明代码，而非具体的实现。
在版权法的语境下，这一性质导致合理使用更易适用于 API。
类似于 QWERTY 键盘布局方式，通过按特定的键会调用特定的字母。
如果 QWERTY 键盘布局都被算作专利来保护，那么它们就控制了所有的打字机。

十五、合理使用因素二：使用的目的和性质
Breyer 表示，谷歌使用并改造 Java API 的行为，其目的在于扩展基于安卓系统的智能手机的功能和实用性。
这种行为创造了一个能让程序员轻松使用的新平台。
谷歌对 Java API 的使用仅以实现智能手机程序的实用性所需为限。

十六、合理使用因素三：被复制受保护内容的数量和实质性
Breyer 认为，谷歌只使用了大约千分之四的 Java 源代码，可忽略不计。
针对实质性问题，他认为谷歌没有复制 Java 实现功能所需的核心代码。
谷歌复制这些代码，不是因为它们多么有创造力或多么美观，甚至某种程度上也不是因为它们自身的目的。
谷歌复制这些代码，只是因为程序员已经习惯了使用 Java SE；如果没有它们，将很难吸引程序员使用安卓系统。

十七、合理使用因素四：复制行为的市场效应
Breyer 认为，谷歌在复制 Java API 时，尚不清楚安卓系统是否会成功。
安卓系统不应被视为 Java 的替代品，而应为在不同平台上运行的产品。
Breyer 进一步表示，如果判决支持甲骨文，可能会对公众造成威胁，因为只有甲骨文将掌握关键内容。
这一结果很可能为甲骨文或其他拥有计算机接口版权的公司带来极高的利益。
但是这种控制力与作为版权法基础的激励创造的目的相冲突，而不是进一步助益该目标。
Breyer 就此写道：谷歌对 API 的使用满足所有四个因素的要求。
谷歌的使用只是允许用户将他们积累的才能用于新的转换性平台所必须的。
既然谷歌的复制行为构成合理使用，没有违反版权法，那么认定 API 是否受版权保护亦不再必要。

十八、两个案例的启示
从以上两个案例中可以初步了解到，任何一项软件业务活动都会受到规制规则的制约。
这些规则或法律是软件工程活动或软件业务生存环境的重要组成部分之一。
学习软件工程经济学，必须了解本领域相关法律与规则的基础知识。

十九、本章小结
软件业务活动或软件项目的绩效与规则、规制密切相关。
除采用财务盈利规则的财务分析与评价之外，非财务盈利规则下的经济分析方法包括：
经济费用效益分析法
费用效果分析法
这两种方法各有适用条件及其范围。
在软件项目或软件业务的经济分析中，需要正确分析和评价：
软件与环境的关系
软件与社会的关系
软件与治理的关系
软件业务规制可分为：
经济性规制
社会性规制
竞争
规则和规制是软件业务活动或软件工程活动经济分析与评价的重要内容之一。
学习软件工程经济学，必须了解相关法律与规则的基础知识。
